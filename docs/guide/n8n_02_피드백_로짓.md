## 목표

유저가 식당 카드에서 👍(붐업) 또는 👎(붐다운)을 누르면  
해당 카테고리의 로짓이 **±0.2** 즉시 업데이트됩니다.

### 수정 전 흐름
```
피드백 입력(Webhook) → 음식 피드백(Supabase Insert) → Respond to Webhook
```

### 수정 후 흐름
```
피드백 입력(Webhook)
    → 음식 피드백(Supabase Insert)    ← 기존 유지
    → [피드백 현재 로짓](Supabase)    ← 신규 추가
    → [피드백 로짓 계산](Code)        ← 신규 추가
    → [피드백 로짓 저장](HTTP)        ← 신규 추가
    → Respond to Webhook
```

## 2-3. 피드백 현재 로짓 노드 추가

`음식 피드백` 노드 오른쪽 **+** → `Supabase` 추가

| 항목 | 값 |
|------|-----|
| **Operation** | Get Many |
| **Table** | `user_preference_logits` |
| **Return All** | ✅ 체크 |

**Filters:**
```
Key Name  : user_id
Condition : equal
Key Value : {{ ($('피드백 입력').item.json.body || $('피드백 입력').item.json).user_id }}
```

> 💡 `body.user_id`와 `user_id` 두 가지 형태 모두 처리하는 표현식입니다.  
> n8n webhook이 body를 어떻게 파싱하느냐에 따라 다를 수 있어서 둘 다 처리합니다.

노드 이름: **`피드백 현재 로짓`**

## 2-4. 피드백 로짓 계산 노드 추가

`피드백 현재 로짓` 오른쪽 **+** → `Code` 추가

```javascript
// ────────────────────────────────────────────────
// 피드백 로짓 계산 노드
// ────────────────────────────────────────────────
// 목적:
//   유저가 👍(like) 또는 👎(dislike)를 눌렀을 때
//   해당 카테고리의 로짓을 ±0.2 조정
//
// 입력:
//   - $('피드백 입력'): webhook body (user_id, category, feedback)
//   - $input.all()   : 피드백 현재 로짓 노드의 결과 (현재 로짓 값들)
//
// 출력:
//   - 업데이트할 단 1개의 row (user_id, category, 새 logit, ...)
// ────────────────────────────────────────────────

// 피드백 입력 웹훅에서 데이터 꺼내기
// (n8n 버전에 따라 body가 있을 수도 없을 수도 있어서 둘 다 처리)
const raw    = $('피드백 입력').first().json
const body   = raw.body || raw

const userId   = body.user_id
const category = body.category
const feedback = body.feedback   // 'like' 또는 'dislike'

// 학습률 설정
// 명시적 피드백이므로 0.2로 설정 (월드컵 0.5보다는 약하게)
const ALPHA = 0.2
const delta = feedback === 'like' ? ALPHA : -ALPHA

// 현재 카테고리의 로짓 값 가져오기
// DB에 해당 카테고리 row가 없으면 0에서 시작
const allLogits = $input.all()
const existing  = allLogits.find(i => i.json.category === category)

const currentLogit = existing ? (existing.json.logit ?? 0) : 0
const currentCount = existing ? (existing.json.sample_count ?? 0) : 0

// 새 로짓 계산 (소수점 3자리로 반올림)
const newLogit = Math.round((currentLogit + delta) * 1000) / 1000

return [{
  json: {
    user_id:      userId,
    category:     category,
    logit:        newLogit,
    sample_count: currentCount + 1,      // 피드백 횟수 +1
    updated_at:   new Date().toISOString() // 시간 감쇠용 타임스탬프
  }
}]
```

노드 이름: **`피드백 로짓 계산`**

## 2-5. 피드백 로짓 저장 노드 추가 (HTTP Request)

`피드백 로짓 계산` 오른쪽 **+** → `HTTP Request` 추가

> 💡 **왜 Supabase 노드 대신 HTTP Request?**  
> Supabase n8n 노드는 **덮어쓰기(Upsert)** 시 현재 값에 더하는 방식이 아닌  
> 새 값으로 완전 교체하는 방식입니다.  
> HTTP Request로 REST API를 직접 호출하면 `Prefer: resolution=merge-duplicates` 헤더로  
> 안전하게 Upsert를 처리할 수 있습니다.

### HTTP Request 설정

| 항목 | 값 |
|------|-----|
| **Method** | POST |
| **URL** | `https://[프로젝트ID].supabase.co/rest/v1/user_preference_logits` |
| **Authentication** | Predefined Credential Type |
| **Credential Type** | Supabase API |
| **Credential** | 기존 Supabase account 선택 |

> ⚠️ URL의 `[프로젝트ID]` 부분을 실제 Supabase 프로젝트 ID로 교체하세요.  
> 예: `https://hauqlkowokezfxcgejzp.supabase.co/rest/v1/user_preference_logits`

### Headers (Send Headers 켜기)

| Name | Value |
|------|-------|
| `Prefer` | `resolution=merge-duplicates` |
| `Content-Type` | `application/json` |

### Body

- **Body Content Type**: Raw
- **Content Type**: `application/json`
- **Body**:

```
={{ JSON.stringify([$json]) }}
```

> 💡 `[$json]`처럼 배열로 감싸는 이유:  
> Supabase REST API는 배열 형태의 JSON을 받아야 합니다.

노드 이름: **`피드백 로짓 저장`**

`피드백 로짓 저장` 오른쪽 **+** → `Respond to Webhook` 추가 (이미 있으면 연결만)

**Response Body** (이미 있는 노드를 재사용하는 경우 설정 확인):
```json

```

## 2-7. 전체 흐름 확인

```
피드백 입력(Webhook)
    ↓
음식 피드백(Supabase - Create, food_feedback 테이블)   ← 기존 그대로
    ↓
피드백 현재 로짓(Supabase - Get Many, user_preference_logits)
    ↓
피드백 로짓 계산(Code - ±0.2 계산)
    ↓
피드백 로짓 저장(HTTP Request - Supabase Upsert)
    ↓
Respond to Webhook
```

**Save → Active 확인**

## 🧪 테스트

### 붐업 테스트 (like)
```bash
POST http://n8n-host:5678/webhook/feedback
Content-Type: application/json

{
  "user_id": "test_user_001",
  "restaurant_id": "some-restaurant-uuid",
  "food_name": "돈까스",
  "category": "분식",
  "feedback": "like"
}
```

**확인:**
- Supabase Table Editor → `user_preference_logits`
- `user_id=test_user_001`, `category=분식` 행의 `logit` 값 확인
- 처음 요청: `logit = 0.2`
- 한 번 더: `logit = 0.4`
- 세 번: `logit = 0.6`

### 붐다운 테스트 (dislike)
```bash
POST http://n8n-host:5678/webhook/feedback
Content-Type: application/json

{
  "user_id": "test_user_001",
  "restaurant_id": "some-restaurant-uuid",
  "food_name": "파스타",
  "category": "양식",
  "feedback": "dislike"
}
```

**확인:**
- `category=양식` 행의 `logit = -0.2`
