# 변경 이력 — 2026-05-19

## 1. 피드백 버튼 UI 개선 (`web/src/pages/Home.tsx`)

**변경 내용:**  
식당 카드의 👍/👎 버튼에서 선택된 버튼을 시각적으로 강조.

- 선택된 버튼: 배경 밝아짐 + 글자 색 밝아짐 + 크기 13→15px + glow 효과 (`box-shadow`)
- 비선택 버튼: 평소 상태 그대로 유지 (어둡게 처리 없음)
- `transition: all 0.2s` 추가 (부드러운 전환)
- 한 번 클릭 후 재클릭 방지: `disabled={!!voted}` + `if (voted) return` 가드 유지

```tsx
// like 선택 시
boxShadow: voted === 'like' ? '0 0 8px #4caf5088' : 'none'
fontSize: voted === 'like' ? 15 : 13

// dislike 선택 시
boxShadow: voted === 'dislike' ? '0 0 8px #ff6b6b88' : 'none'
fontSize: voted === 'dislike' ? 15 : 13
```

## 2. 피드백 필드명 통일 (`web/src/lib/n8n.ts`)

**변경 내용:**  
`sendFeedback()`이 n8n에 POST할 때 키 이름을 `feedback` → `reaction`으로 변환.

**배경:**  
- Supabase `food_feedback` 테이블 컬럼명: `reaction`
- n8n `음식 피드백` 노드 필드명: `reaction`
- React 내부 변수명: `feedback` (유지)

TypeScript 파라미터명은 `feedback`으로 유지하되, POST body에서만 `reaction`으로 바꿔 전송.

```typescript
const { feedback, ...rest } = params
await axios.post(FEEDBACK_WEBHOOK, { ...rest, reaction: feedback }, { timeout: 5000 })
```

## 4. 메뉴 조회 구조 개편 (`web/src/lib/n8n.ts`, `web/src/pages/Home.tsx`)

**배경:**  
`음식_상세` 워크플로우 제거로 `/webhook/food/menu` endpoint 소멸.  
대신 음식 추천 응답(A/B/C 경로)에 menus를 포함시키는 구조로 전환.

**변경 내용:**

- `Restaurant` interface에 `menus?: MenuItem[]` 추가
- `fetchRestaurantMenu()` 함수 제거 (별도 API 호출 불필요)
- `handleMenuRequest` → async API 호출 제거, `restaurant.menus` 직접 사용

```typescript
// 변경 전
async function handleMenuRequest(restaurant: Restaurant) {
  setMenuState({ ..., loading: true })
  const result = await fetchRestaurantMenu({ restaurant_id: restaurant.restaurant_id })
  setMenuState({ ..., menus: result.menus, loading: false })
}

// 변경 후
function handleMenuRequest(restaurant: Restaurant) {
  setMenuState({ restaurant, menus: restaurant.menus ?? [], loading: false, ... })
}
```
